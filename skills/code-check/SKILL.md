# Code-Check

Wenn du selbst etwas gebaut hast, willst du wissen: Funktioniert das zuverlässig? Ist das sicher genug? Was kann besser werden? Dieser Skill prüft dein Werk und erklärt die Ergebnisse so, dass auch Nicht-Entwickler:innen sie verstehen — mit klaren Prioritäten und konkreten „so machst du es besser"-Hinweisen. Dein Sicherheitsnetz beim Citizen Coding.

## Auslöser — wann ich genutzt werde
Nutze diesen Skill für: Eigenes Tool prüfen · Vor dem Teilen checken · Verständliche Verbesserungen.
Stichworte: bauen, qa, citizen-coding, review, qualität.

## Vorgehen
1. Eingabe/Kontext erfassen (Datei, Text oder Auftrag).
2. Die Anforderung in kleine Schritte zerlegen und umsetzen.
3. Ergebnis strukturiert erzeugen (siehe Ausgabe).
4. Kurz zusammenfassen, was getan wurde.

## Ausgabe
Lauffähige Datei (HTML/Tool), die du direkt öffnen und prüfen kannst.

## Regeln
- Klar und für Nicht-Techniker:innen verständlich bleiben.
- Nichts erfinden — nur, was aus der Eingabe hervorgeht.
- Bei Unklarheit kurz nachfragen statt raten.
