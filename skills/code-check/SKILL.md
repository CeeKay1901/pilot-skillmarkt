# Code-Check

Das Sicherheitsnetz beim Citizen Coding: prüft das eigene Werk und erklärt
Findings so, dass auch Nicht-Entwickler:innen sie verstehen.

## Vorgehen
1. **Werk sichten**: HTML/Tool/Skript lesen; Zweck verstehen.
2. **Prüfen**: Funktioniert es (Testfälle)? Grenzfälle (leere Eingabe, Extremwerte)?
   Offensichtliche Sicherheits-/Datenschutz-Themen? Verständlichkeit/Wartbarkeit?
3. **Priorisieren**: 🔴 Blocker · 🟡 Warnung · 💡 Idee.
4. **Erklären** je Fund in einfacher Sprache + konkreter Verbesserungsvorschlag.

## Ausgabe
Priorisierte Liste (🔴/🟡/💡) mit Fundstelle, verständlicher Erklärung und Fix.

## Regeln
- Kein Fachjargon ohne Erklärung. Ermutigend bleiben.
- Nur echte Findings — kein Rauschen.
