# Prototyp bauen

Für Kolleg:innen ohne Programmier-Vorkenntnisse: aus einer Idee wird ein
lauffähiges Tool, das sofort im Browser läuft.

## Vorgehen
1. **Verstehen** — Was soll das Tool tun? Welche Eingaben, welches Ergebnis?
   Bei Unklarheit 1–2 gezielte Rückfragen stellen.
2. **Bauen** — eine eigenständige HTML-Datei (inline CSS/JS), im pilot-Design.
3. **Öffnen** — Datei im Browser anzeigen, damit man es sofort sieht.
4. **Iterieren** — "Was soll anders sein?" → anpassen, bis es passt.

## Regeln
- Alles in **einer** Datei → per Doppelklick teilbar.
- Reduziert & bedienbar, kein Fachjargon in der Oberfläche.
- Realistisch bleiben: kleine Tools, keine komplexen Systeme.

## Bausteine
Siehe `references/muster.md` für erprobte Muster (Rechner, Formular,
Tabelle) und `scripts/berechnung.js` als Vorlage für Live-Berechnung.
