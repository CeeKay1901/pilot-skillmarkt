Ihr kennt KI bisher als Chat: fragen, Antwort bekommen. Ein Skill ist der nächste Schritt — ein gespeicherter Arbeitsablauf, den die KI für dich ausführt. Zuhause ist er in Claude Code, viele laufen aber auch in Langdock. Und ihr könnt damit selbst Dinge bauen.

Langdock & Claude Code

Zwei Werkzeuge,
zwei Stärken

Langdock ist bei uns zu Recht gesetzt: die Top-Modelle im Chat, eigene Assistenten fürs Team, Workflows, euer Firmenwissen angebunden — DSGVO-konform in der EU gehostet. Claude Code nimmt davon nichts weg. Es beginnt dort, wo aus einer guten Antwort ein fertiges Ergebnis werden soll: eine Datei, ein Tool, eine Webseite, die du behältst und weiterentwickelst.

LangdockKI-Plattform · im Alltag gesetzt
Claude CodeKI-Agent · der nächste Schritt

Im Browser: chatten, Team-Assistenten nutzen, Workflows starten — und seit Kurzem eigene Skills hochladen.
Per Auftrag: es arbeitet selbstständig direkt in deinen Dateien und Ordnern.
Was rauskommt
Starke Antworten, Texte und Analysen — auf Wunsch auch als PDF-, Word- oder Excel-Datei.
Lauffähige Ergebnisse: Tools, Websites, Reports, Automationen.

Bauen und Weiterentwickeln — alles, was als Datei existieren und wachsen soll.

DSGVO-konform, EU-Hosting, ISO-zertifiziert — genau dafür ist es gesetzt.
Arbeitet lokal auf deinem Rechner — deine Dateien bleiben bei dir.

Volle Transparenz

Was steckt drin?

Kein Blackbox-Zauber. Ein Skill ist einfach ein Ordner mit einer Textdatei — der SKILL.md. Darin steht ein kurzer Steckbrief und die Anleitung, die Claude Code befolgt. Du kannst bei jedem Skill reinschauen, ihn verstehen und als Vorlage für eigene nehmen.

webaudit/
≡ SKILL.md
≡ references/checkliste.md
Ordner = Skill.
Die SKILL.md ist das Herz.
Steckbrief (Frontmatter)
---
name: webaudit
description: Website-Check — UX, Speed, SEO, A11y
trigger: /webaudit
---
Anleitung (was Claude tut)
# webaudit

Prüfe die Seite in zwei Stufen:
1. Code scannen (SEO-Tags, Alt-Texte, Links)
2. Im echten Browser öffnen, Screenshots machen

Gib einen priorisierten Report aus:
- 🔴 Kritisch  - 🟡 Empfehlung

In jedem Skill findest du den Tab „Dateien & Download“ — schau rein.

Auslöser

Zwei Wege, wie ein Skill startet

Du musst den Befehl gar nicht kennen. Ein Skill kann auf zwei Arten anspringen — beides gleichzeitig möglich.

1
Manuell — du rufst ihn

Du tippst den Befehl direkt. Klar und bewusst.

› /webaudit
2
Automatisch — Claude erkennt ihn

Du schreibst ganz normal, was du brauchst. Claude erkennt selbst, welcher Skill passt — und startet ihn ohne Befehl.

„Prüf mal, ob mit unserer Landingpage alles stimmt.“
So entscheidet Claude

Claude kennt von allen Skills nur den Namen und die eine description-Zeile — wie eine Speisekarte. Passt deine Aufgabe dazu, lädt es den kompletten Skill und wendet ihn an. Darum ist die description das Erkennungsmerkmal; für heikle Aktionen lässt sich der Automatik-Start bewusst abschalten.

Die größeren Bausteine

Vom Skill zum Framework

Im Katalog begegnen dir neben Skills noch zwei größere Bausteine — gleiches Prinzip, eine Nummer größer. Bleiben wir beim Bild vom Rezept:

Skill — eine Fähigkeit

Ein Ablauf in einer Textdatei, den du per /befehl oder ganz normalem Satz abrufst. Wie ein einzelnes Rezept.

Plugin — ein Paket

Bündelt mehrere zusammengehörige Skills und Befehle und kommt mit einem einzigen /plugin-Befehl in dein Claude Code. Wie ein Kochbuch: einmal ins Regal gestellt, alle Rezepte griffbereit.

Framework — eine Arbeitsweise

Kein Download, sondern ein komplettes Vorgehen, das deine Zusammenarbeit mit Claude strukturiert — von der Idee über den Plan bis zum geprüften Ergebnis. Wie eine Art zu kochen, nicht nur ein Rezept.

Im Katalog filterst du oben über die Typ-Leiste: Skills · Plugins · Frameworks — Plugins und Frameworks tragen ihren Typ als Badge auf der Karte.
