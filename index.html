Was ist
ein Skill?

Ihr kennt KI bisher als Chat: fragen, Antwort bekommen. Ein Skill ist der nächste Schritt — ein gespeicherter Arbeitsablauf, den die KI für dich ausführt. Zuhause ist er in Claude Code, viele laufen aber auch in Langdock. Und ihr könnt damit selbst Dinge bauen.

Die Kurzfassung

Ein Skill ist wie ein Rezept.

Ein gutes Rezept hält einen erprobten Ablauf fest, damit das Gericht jedes Mal gelingt — ohne dass du bei null anfängst. Ein Skill macht genau das für Arbeit am Rechner: Ein:e Kolleg:in hat einen sinnvollen Ablauf einmal sauber festgehalten, und du rufst ihn per Befehl ab.

Rezept
Zutaten + Schritte → immer gleiches Ergebnis
≈
Skill
/befehl → Claude erledigt den Ablauf für dich

Langdock & Claude Code

zwei Stärken

Langdock ist bei uns zu Recht gesetzt: die Top-Modelle im Chat, eigene Assistenten fürs Team, Workflows, euer Firmenwissen angebunden — DSGVO-konform in der EU gehostet. Claude Code nimmt davon nichts weg. Es beginnt dort, wo aus einer guten Antwort ein fertiges Ergebnis werden soll: eine Datei, ein Tool, eine Webseite, die du behältst und weiterentwickelst.

Ein Skill, zwei Systeme

Wo Skills bei uns laufen

Das Schöne: Im Kern ist ein Skill eine Anleitung in einer Textdatei (SKILL.md) — und die versteht inzwischen mehr als ein System. Jeder Skill im Katalog trägt Chips, die zeigen, wo er einsetzbar ist. Hier der Unterschied:

Das Zuhause der Skills. Hier ist ein Skill ein Ordner mit SKILL.md und optionalen Scripts — Claude Code führt ihn aus und arbeitet dabei direkt in deinen Dateien.

So kommt er rein

ZIP aus dem Katalog laden und nach .claude/skills/ entpacken — oder einfach den /befehl tippen.

Das kann er

Alles: Dateien lesen & schreiben, Scripts ausführen, Browser steuern, Tools und Webseiten bauen, Automationen.

Grenzen

Läuft auf deinem Rechner im Terminal — dafür gibt es den Skill /erste-schritte als Einstieg.

Langdock hat ein eigenes Skills-Feature — gleiches Prinzip, gleiche Datei. Du lädst die SKILL.md hoch, und Langdock aktiviert sie automatisch, sobald deine Nachricht dazu passt. Kein Terminal nötig.

So kommt er rein

In Langdock: Skills → Add Skill → SKILL.md (oder ZIP) hochladen. Teilen mit dem Team geht per Share-Dialog, Admins können Workspace-weit freigeben.

Das kann er

Alles Instruktive: Checklisten, Prüf-Regeln, Templates, Schreib-Workflows. Dateien erzeugt Langdock über seine eigenen System-Skills (PDF, Word, Excel, PowerPoint).

Grenzen

Skills sind hier reine Anleitung: keine eigenen Scripts, kein Dateisystem, kein Browser. Bau-Skills wie webseite-bauen bleiben deshalb Claude-Code-Sache.

Volle Transparenz

Was steckt drin?

Kein Blackbox-Zauber. Ein Skill ist einfach ein Ordner mit einer Textdatei — der SKILL.md. Darin steht ein kurzer Steckbrief und die Anleitung, die Claude Code befolgt. Du kannst bei jedem Skill reinschauen, ihn verstehen und als Vorlage für eigene nehmen.

webaudit/
≡ SKILL.md
≡ references/checkliste.md
Ordner = Skill.
Die SKILL.md ist das Herz.
Steckbrief (Frontmatter)
---
name: webaudit
description: Website-Check — UX, Speed, SEO, A11y
trigger: /webaudit
---
Anleitung (was Claude tut)
# webaudit

Prüfe die Seite in zwei Stufen:
1. Code scannen (SEO-Tags, Alt-Texte, Links)
2. Im echten Browser öffnen, Screenshots machen

Gib einen priorisierten Report aus:
- 🔴 Kritisch  - 🟡 Empfehlung

In jedem Skill findest du den Tab „Aufbau & Dateien“ — schau rein.

Auslöser

Zwei Wege, wie ein Skill startet

Du musst den Befehl gar nicht kennen. Ein Skill kann auf zwei Arten anspringen — beides gleichzeitig möglich.

1
Manuell — du rufst ihn

Du tippst den Befehl direkt. Klar und bewusst.

› /webaudit
2
Automatisch — Claude erkennt ihn

Du schreibst ganz normal, was du brauchst. Claude erkennt selbst, welcher Skill passt — und startet ihn ohne Befehl.

„Prüf mal, ob mit unserer Landingpage alles stimmt.“
Deine Anfrage „Prüf mal, ob mit unserer Landingpage alles stimmt.“
Claude vergleicht mit der Skill-Liste ↓
webauditprüft Websites auf UX, Speed, SEO & BarrierefreiheitMatch
briefing-generstellt strukturierte Briefings
daten-aufbereitenbereinigt Excel- & CSV-Dateien
↓
webaudit startet automatisch — ganz ohne /befehl.
So entscheidet Claude

Claude hat von allen Skills immer nur den Namen und die eine description-Zeile im Kopf — wie eine Speisekarte. Passt deine Aufgabe dazu, lädt Claude den kompletten Skill und wendet ihn an. Das hält alles schnell, auch bei vielen Skills.

Darum ist die description in der SKILL.md so wichtig: Sie ist das Erkennungsmerkmal. Für heikle Aktionen (z. B. etwas veröffentlichen) lässt sich der Automatik-Start bewusst abschalten — dann zählt nur der Befehl.
